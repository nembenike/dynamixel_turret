# Installation Guide

This guide covers system packages, Python dependencies, build prerequisites, and verification steps for Dynamix Turret.

## Supported environment

Primary target is Linux with:

- GNU toolchain (`g++`, `make`)
- Video device support for OpenCV (typically V4L2)
- Access to a serial USB Dynamixel adapter

## Required components

### Native build/runtime

- C++ compiler with GNU++20 support
- `raylib` development package
- OpenCV development headers/libraries (`core`, `videoio`, `imgproc`)
- Dynamixel SDK C library (`libdxl_x64_c`)
- Standard X11-related libs (already referenced by the build)

### Python runtime

From [requirements.txt](../requirements.txt):

- `mediapipe>=0.10.14`
- `opencv-python>=4.9.0`

Install with:

- `python3 -m pip install -r requirements.txt`

## Build configuration source

The main build command is in [Makefile](../Makefile):

- Compiler flags from `CXXFLAGS`
- Include search paths from `INCLUDE`
- Link search paths from `LIBDIR`
- Runtime libs from `LIBS`

If packages are installed in non-default locations, update these variables or pass them inline when invoking `make`.

## Linux permissions for serial devices

The turret must open `/dev/tty*` devices. Typical setup:

- Add user to serial-access group (often `dialout`)
- Re-login after group changes
- Avoid permanent world-writable serial nodes in production

`make run` performs a best-effort temporary chmod on the selected port.

## Build and run

1. Install Python dependencies:
   - `make install-python-deps`
2. Build binary:
   - `make`
3. Launch:
   - `make run`

Binary output is `./turret`.

## First-run verification checklist

1. Start the app and confirm the window opens.
2. Confirm title/status area shows an active serial port/baud.
3. Move with mouse in the aim pad and verify servo motion.
4. Press `H` and confirm hand mode state switches.
5. Complete calibration and verify hand movement maps to pan/tilt.

## MediaPipe model behavior

On some Python/MediaPipe builds, the hand-tracking process may use MediaPipe Tasks mode and download a model to:

- `~/.cache/dynamix_turret/hand_landmarker.task`

Override with environment variable `HAND_LANDMARKER_MODEL_PATH`.

## Optional performance tuning

- Reduce frame traffic with `HAND_FRAME_SEND_EVERY` (for example, 2 or 3)
- Use a lower camera resolution only if you also update protocol assumptions in C and Python
- Ensure no competing camera consumers run simultaneously

## Clean artifacts

Run:

- `make clean`

This removes the binary and Python cache artifacts generated by this project.
